# 架构说明

本项目按战略情报系统拆分为七层：

| 层级 | 目录 | 职责 |
|---|---|---|
| 执行规则 | `strategy-monitor/` | 定义 skill、prompt、输入约束和报告模板 |
| 信息来源 | `sources/` | 固定来源、采集周期、检索问题 |
| 证据沉淀 | `cache/` | 人类可读证据、竞品画像、客户缺口 |
| 结构化数据 | `data/` | JSONL 事实、证据、评分、行动项 |
| 商业逻辑 | `logic/` | H1-H6、评分、止损、竞品模型 |
| 输出报告 | `reports/` | 周/月/季/半年/年度和专题报告 |
| 展示 | `viewer/` | 本地静态浏览、搜索、筛选和关联 |

核心原则：证据进入 `cache/` 和 `data/` 后可被持续复用；战略判断必须经过 `logic/`，不能只依赖单次搜索。
