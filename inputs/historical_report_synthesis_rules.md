# 历史报告综合规则

用于月度复盘、季度战略监控或用户要求“综合前几个月报告”时，把 `output/` 中既有报告纳入本期判断。历史报告只能作为连续样本和管理闭环证据，不能替代本期外部证据核验。

## 读取范围

默认读取：

- 最近 3 份 `output/monthly-*-small-power-charging-strategy-review.md`。
- 最近 1 份 `output/*-evidence-matrix.md`。
- 最近 1 份 `output/*-action-tracker.md`。
- 若存在上一季度报告，读取最近 1 份 `output/quarterly-*-small-power-charging-strategy-review.md`。

如果文件不足 3 份，必须在报告中标注“历史样本不足”，并只对已有样本做趋势判断。

## 提取字段

从历史报告中提取：

- H1-H6 每期状态：`成立`、`部分变化`、`不成立`、`风险上升`、`高风险`、`证据不足`。
- 每期最终决策：`维持战略`、`小幅调整`、`重大调整`、`立即止损`。
- 高风险信号及其是否重复出现。
- 证据等级变化：`弱`、`中`、`强`、`内部必需`、`证据不足`。
- 未关闭的内部数据缺口。
- 上期 1-4 周行动的完成状态。
- 被重复提出但没有关闭的管理问题。

## 综合判断规则

| 条件 | 处理方式 |
|---|---|
| 同一风险连续 3 期出现 | 升级为季度级风险 |
| 同一内部数据缺口连续 2 期未关闭 | 标记为管理闭环问题 |
| 同一结论证据等级从 `弱` 升级到 `中` 或 `强` | 可提高战略判断置信度 |
| 同一结论证据等级连续未提升 | 维持观察，不得升级为强结论 |
| 上期 P0/P1 行动未完成 | 本期必须降低相关策略置信度，或升级责任闭环 |
| H1-H6 中任一假设连续 2 期从 `成立` 变为 `部分变化` 或 `高风险` | 触发季度战略复核 |
| H6 连续 2 期为 `高风险` | 必须讨论止损红线、项目冻结或客户结构调整 |

## 输出要求

季度报告必须包含：

- 历史报告清单。
- H1-H6 历史状态趋势表。
- 证据等级变化表。
- 未关闭内部数据缺口表。
- 上期行动闭环表。
- 本期新增证据与历史结论的差异。
- 是否触发季度级战略调整。

不得把历史报告中的旧结论直接复制为本期结论。每个本期关键判断都必须说明：

```text
历史结论：
本期新增证据：
趋势变化：
内部数据缺口：
本期判断：
```
